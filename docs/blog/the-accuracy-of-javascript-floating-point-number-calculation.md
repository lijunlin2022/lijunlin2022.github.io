# JavaScript 浮点数计算的精度问题

## 1 现象

当 JavaScript 的浮点数做数学运算时，时常会出现一些问题：

```javascript
const sum1 = 0.1 + 0.2
console.log(sum1) // 0.30000000000000004

const sum2 = 0.15 + 0.15
console.log(sum2) // 0.3

const diff = 0.3 - 0.2
console.log(diff1) // 0.09999999999999998

const product1 = 19.9 * 10 * 10
console.log(product1) // 1990

const product2 = 19.9 * 100
console.log(product2) // 1989.9999999999998

const quotient = 0.3 / 0.1
console.log(quotient) // 2.9999999999999996
```

## 2 原因

JavaScript 采用了 IEEE754 标准来规定数字

IEEE 754 标准如下，其中未划线的就是 JavaScript 采用的标准

* ~~单精度~~
* 双精度 (64 为)
* ~~延伸单精度~~
* ~~延伸双精度~~

64 位分为三部分，分别是`符号位`（1 位）、`指数位`（11 位）和`有效位`（52 位）

* 符号位，0 标识正数，1 表示负数
* 指数位用来存储科学记数法的指数，也就是 n 次方
* 有效位表用二进制表示

举例而言：十进制 0.125 转二进制

```
0.125 × 2 = 0.25, 整数部分为 0
0.25 × 2 = 0.5, 整数部分为 0
0.5 × 2 = 1.0, 整数部分为 1

// 0.125 -> 0.001
```

下面我们来看十进制的 0.1 和 十进制 0.2 的加法

```
0.1 × 2 = 0.2，整数部分是 0
0.2 × 2 = 0.4，整数部分是 0
0.4 × 2 = 0.8，整数部分是 0
0.8 × 2 = 1.6，整数部分是 1
0.6 × 2 = 1.2，整数部分是 1
0.2 × 2 = 0.4，整数部分是 0
...

// 0.1 -> 0.00 0110 0110 0110 0110 ...
```

```
0.2 × 2 = 0.4，整数部分是 0
0.4 × 2 = 0.8，整数部分是 0
0.8 × 2 = 1.6，整数部分是 1
0.6 × 2 = 1.2，整数部分是 1
0.2 × 2 = 0.4，整数部分是 0
...

// 0.2 -> 0.00 1100 1100 1100 1100 ...
```

IEEE 754 标准的双精度浮点数的小数部分最多支持 52 位二进制位，所以二者相加得到

```
  0.00 0110 0110 0110 0110 0110 0110 0110 0110 0110 0110 0110 0110 01
+ 0.00 1100 1100 1100 1100 1100 1100 1100 1100 1100 1100 1100 1100 11
----------------------------------------------------------------------
  0.01 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 00
```

然后再将 `0.01 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 00` 转变为十进制就成为了 `0.30000000000000004`

## 3 应用场景

### 3.1 如何判断两个浮点数是否相等

精度判断法：

```javascript
const delta = 1e-5  // 精度精确到 0.00001
const num1 = 0.1
const num2 = 0.2
const sum = 0.3

// 判断相等小于精度就认为相等
if (num1 + num2 - sum < delta) {
    console.log('a + b === sum')
}
```

### 3.2 toFixed() 方法的坑

JavaScript 可以使用 `toFixed()` 方法定点表示一个数值，原则大部分简单称为四舍五入，但实际情况有些复杂：

```javascript
1.35.toFixed(1)		// '1.4'，正确
1.335.toFixed(2)	// '1.33'，错误
1.3335.toFixed(3)	// '1.333'，错误
```

且 `toFixed()` 在不同浏览器上的结果不一定相同，这一点需要注意

## 4 解决办法

思路：将小数转变为整数，计算后再将结果转变为小数

### 4.1 简单的浮点数加法

这个题目我在面试的时候遇到过，当时使用奇技淫巧强行 AC 的，现在来写一下正确代码

```javascript
function add (num1, num2) {
  const str1 = num1.toString()
  const str2 = num2.toString()
  let len = 0
  let len1 = 0
  let len2 = 0
  // 如果 num1 是浮点数
  if (str1.includes('.')) {
    len1 = str1.split('.')[1].length
  }
  // 如果 num2 是浮点数
  if (str2.includes('.')) {
    len2 = str2.split('.')[1].length
  }
  len = Math.max(len1, len2)
  const base = Math.pow(10, len)
  const iNum1 = num1 * base
  const iNum2 = num2 * base
  return (iNum1 + iNum2) / base
}
```
