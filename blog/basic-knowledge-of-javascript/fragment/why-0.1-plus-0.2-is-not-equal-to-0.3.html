为什么 0.1 + 0.2 !== 0.3 #

1 问题描述 #

JavaScript 做浮点数的数学运算时，时常会出现一些问题：

javascript
const sum1 = 0.1 + 0.2
console.log(sum1) // 0.30000000000000004

const sum2 = 0.15 + 0.15
console.log(sum2) // 0.3

const diff = 0.3 - 0.2
console.log(diff) // 0.09999999999999998

const product1 = 19.9 * 10 * 10
console.log(product1) // 1990

const product2 = 19.9 * 100
console.log(product2) // 1989.9999999999998

const quotient = 0.3 / 0.1
console.log(quotient) // 2.9999999999999996

2 简要解释 #

根本原因是：计算机使用二进制来表示浮点数，而一些浮点数用二进制无法完整表示，因此计算机只能够将这些浮点数截断后处理。

浮点数截断之后，再进行加减乘除就会出现误差。

有的语言本身提供了消除误差的方法，比如 Java。而有的语言没有提供消除误差的方法，比如 JavaScript。

3 详细解释 #

JavaScript 采用了 IEEE 754 标准中的双精度（64 位）来规定数字

64 位分为三部分，分别是符号位（1 位）、指数位（11 位）和有效位（52 位）

  • 符号位，0 标识正数，1 表示负数
  • 指数位用来存储科学记数法的指数，也就是 n 次方
  • 有效位用二进制表示

简单起见，我们现在只关注有效位。以十进制 0.1 + 0.2 为例，我们先把它转成二进制计算，得到结果后再转回十进制。

// 十进制的 0.1 转二进制

0.1 × 2 = 0.2，整数部分是 0
0.2 × 2 = 0.4，整数部分是 0
0.4 × 2 = 0.8，整数部分是 0
0.8 × 2 = 1.6，整数部分是 1
0.6 × 2 = 1.2，整数部分是 1
0.2 × 2 = 0.4，整数部分是 0
...

// 0.1 -> 0.00 0110 0110 0110 0110 ...

// 十进制的 0.2 转二进制

0.2 × 2 = 0.4，整数部分是 0
0.4 × 2 = 0.8，整数部分是 0
0.8 × 2 = 1.6，整数部分是 1
0.6 × 2 = 1.2，整数部分是 1
0.2 × 2 = 0.4，整数部分是 0
...

// 0.2 -> 0.00 1100 1100 1100 1100 ...

IEEE 754 标准的双精度浮点数的小数部分最多支持 52 位二进制位，所以二者相加得到

  0.00 0110 0110 0110 0110 0110 0110 0110 0110 0110 0110 0110 0110 01
+ 0.00 1100 1100 1100 1100 1100 1100 1100 1100 1100 1100 1100 1100 11
----------------------------------------------------------------------
  0.01 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 00

然后再将 0.01 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 0011 00 转变为十进制就成为了 0.30000000000000004

4 解决办法 #

最简单有效的方法是引入 JavaScript 的数学库，比如 math.js。

如果迫不得已，一定要使用 JavaScript 来进行数学计算的话，那么可以使用下面的方法：

它的思路是：将小数转变为整数，计算后再将结果转变为小数。

javascript
const add = (num1, num2) => {
  // 获取 num1 小数点后的位数，如果 num1 是整数，位数则为 0
  const len1 = (String(num1).split('.')[1] || '').length
  // 获取 num2 小数点后的位数，如果 num2 是整数，位数则为 0
  const len2 = (String(num2).split('.')[1] || '').length
  const maxLen = Math.max(len1, len2)
  const base = 10 ** maxLen
  // 将 num1 和 num2 转换为整数，计算完成后再转换为小数
  return ((num1 * base)  + (num2 * base)) / base
}

const sub = (num1, num2) => add(num1, -num2)

5 扩展 #

5.1 如何判断两个浮点数是否相等 #

精度判断法：

javascript
const delta = 1e-5  // 精度精确到 0.00001
const num1 = 0.1
const num2 = 0.2
const sum = 0.3

// 判断相等小于精度就认为相等
if (num1 + num2 - sum < delta) {
    console.log('a + b === sum')
}

5.2 toFixed() 方法的坑 #

我们有时会使用 JavaScript 的 toFixed() 来进行小数的保留，我们一般会预期小数保留时能够四舍五入，但实际情况有些复杂：

javascript
1.35.toFixed(1)		// '1.4'，正确
1.335.toFixed(2)	// '1.33'，错误
1.3335.toFixed(3)	// '1.333'，错误

且 toFixed() 在不同浏览器上的结果不一定相同，所以不建议使用 toFixed 方法。
